# Injekko Architecture Map

This is the shortest useful reading order for the current graph-first, instance-first runtime.

## 1. Authoring graph and blocks
- File: `Editor/GraphToolkit/InjekkoGraphToolkitAuthoring.cs`
- What it does: defines the Graph Toolkit authoring surface.
- Main path: `Bind Declaration -> Instance -> To Type Inferred / To Injectable Type`.
- What matters: the graph only describes explicit scope bindings. It does not describe constructor/member injection internals.

## 2. Importer and lowering into `InjekCompiledScopePlan`
- Files:
  - `Editor/GraphToolkit/InjekkoGraphImporter.cs`
  - `Editor/GraphToolkit/InjekkoGraphImportLowering.cs`
  - `Runtime/Unity/InjekCompiledScopePlan.cs`
- What it does: imports `.injekgraph` into the compiled runtime plan.
- Source of truth rule: after import, `InjekCompiledScopePlan` is the only IR the rest of the package should consume.

## 3. Graph plan codegen
- File: `Editor/InjekGraphPlanCodeGenerator.cs`
- What it does: reads all compiled plans and writes `Assets/Injekko/Generated/InjekkoGraphPlans.Generated.cs`.
- Output shape: one generated method per `GraphId`, each receiving `IInjekGraphReferenceHost` and `InjekScopeNode`.
- Runtime rule: generated graph plans only emit direct `scope.BindInstance(...)` calls in this phase.

## 4. Runtime registry and bootstrap
- Files:
  - `Runtime/Unity/InjekGeneratedRuntimeRegistry.cs`
  - `Runtime/Unity/InjekkoRuntimeBootstrap.cs`
- What it does: registers generated graph-plan delegates and initializes the project root.
- Project rule: `Resources/ProjectPlan.injekgraph` is loaded directly as `InjekCompiledScopePlan`.

## 5. `SceneScope` and `GameObjectScope`
- Files:
  - `Runtime/Unity/SceneScope.cs`
  - `Runtime/Unity/GameObjectScope.cs`
- What they do: act as serialized reference hosts.
- Host rule:
  - `SceneScope` stores scene overrides and scene caches.
  - `GameObjectScope` stores local overrides for a subtree.
  - both call the same generated graph-plan registry at runtime.

## 6. `InjekScopeRegistry`
- File: `Runtime/Unity/InjekScopeRegistry.cs`
- What it does: owns the scope tree.
- Hierarchy rule: `Project -> SceneScope root -> GameObjectScope subscopes`.
- Important: this is the runtime place that decides parent-child scope relationships; it is no longer responsible for discovering graphs or lowering authoring data.

## 7. `_Rizolver` and `_Fucktory`
- Sources:
  - generated by `InjekkoGen`
  - main generator files live under `InjekkoGen/`
- What they do:
  - `_Rizolver` handles activation/injection for `[Injek]` types.
  - `_Fucktory` handles generated creation flows.
- Separation rule: graphs add services to scopes; generated type code activates and resolves dependencies from those scopes.

## One-line mental model
`authoring graph -> imported compiled plan -> generated graph binder -> runtime host -> scope bindings -> generated activation/resolution`
